NEVER share the Spartan heater ground with engine block or signal ground. Heater draws 2A+ during warm-up — this creates massive noise if not isolated.
Power budget @ 5V rail: Teensy 4.1 ~250mA · 74HCT245 ×3 ~150mA · Sensors ~80mA · TJA1051 ×2 ~40mA · Misc ~50mA = ~570mA total. LM2596 rated 3A — comfortable 5× headroom. Spartan 3 Lite runs entirely on its own 12V line — does NOT load the 5V rail at all.
Injector Intercept
Cut ECU injector signal wire. ECU side → IRLZ44N drain. Injector → source. Teensy pin → 100Ω → gate.

Teensy monitors ECU's intended pulse on a digital input, re-fires gate with corrected pulse width referenced to Lambda, MAP, and RPM. Effective ±30% trim range.

1N4007 flyback across each injector is mandatory — coil collapse destroys the MOSFET without it. 100Ω gate resistor damps oscillation.
MAF Signal Synthesis
Verify MAF output type with oscilloscope first — 7A/NF MAF is frequency based (Hz proportional to airflow), not DC voltage.

Teensy reads original MAF frequency on interrupt pin → applies correction factor based on 2.6L vs 2.3L displacement ratio and MAP data → outputs modified frequency to ECU.

ECU "sees" a smaller engine than it is, requesting less fuel. Injector trim layer adds correct fuel back via wideband closed loop.
UART Lambda — LSU ADV ADV
Spartan streams Lambda data continuously at 9600 baud. Teensy reads it asynchronously on Serial1 — no polling delay, no blocking.

The LSU ADV's 20× resolution improvement means the PID loop sees finer grain AFR error. At stoichiometric (λ=1.0) the ADV resolves ~0.0005λ vs ~0.01λ for the 4.9. That granularity directly improves closed-loop correction stability.

No calibration ever required — sensor manufactured to tolerance, no trim resistor needed.
Knock Sensor Signal Conditioning
Knock sensor outputs AC voltage centered at 0V. Teensy ADC is 0–3.3V only — negative voltage will damage it.

Required conditioning:
• 0.1µF coupling cap in series (blocks DC)
• Two 100kΩ to form 1.65V bias (3.3V ÷ 2)
• Signal biased to 1.65V center — safely in ADC range
• 100pF cap to GND for RF noise filtering
At 600MHz, Teensy runs real-time FFT on the knock signal sampled at ~100kHz. Knock frequency on the 2.3/2.6 inline-5 is typically 6–9kHz.

When FFT energy at knock frequency exceeds threshold, Teensy writes modified timing data to the EPROM emulation layer — pulling 1–3° per cycle until knock subsides. Recovery ramp adds timing back at 0.5°/cycle.

Far more precise than the stock Hitachi knock threshold logic.
UART VOLTAGE MISMATCH: Spartan 3 Lite OEM TX pin outputs 5V logic. Teensy 4.1 RX pins are 3.3V tolerant only. A simple 10kΩ/20kΩ voltage divider on the Spartan TX → Teensy RX line is required. Teensy TX → Spartan RX is fine direct — 3.3V reads as logic high on 5V UART input.

BUILD PHILOSOPHY: Each stage is independently testable. Never connect an untested stage to the ECU. Confirm each step with a multimeter or logic analyzer before proceeding. The breadboard prototype validates everything before PCB layout begins.
00

EPROM DUMP — ABSOLUTE FIRST STEP

Before touching anything else, remove the EPROM from the ECU and read it with a TL866II+ or T48 programmer. Save the binary to at least two locations (USB + cloud). This is your recovery file. If the Teensy has a bug and the ECU won't boot, you reflash the original. No exceptions — do this before Stage 01.

01

POWER STAGE — BENCH ONLY

Build LM2596 module + AMS1117 on breadboard. Apply 12V from bench supply. Measure: 5.00V ±0.05V on 5V rail, 3.30V ±0.05V on 3.3V rail. Plug Teensy into 3.3V/5V rails, confirm USB enumeration on laptop. Install Arduino IDE + Teensyduino, flash blink sketch. Solid foundation confirmed.

02

SPARTAN 3 LITE OEM — UART BENCH TEST

Wire Spartan TX → 10kΩ → Teensy RX pin 0, with 20kΩ from that junction to GND. Wire Teensy TX pin 1 → Spartan RX direct. Connect Spartan HTR GND to bench PSU negative before connecting 12V. Confirm Lambda data appears on Serial Monitor at 9600 baud. With sensor in free air expect ~1.0λ or slightly lean. No data = check HTR GND wiring first.

03

LEVEL SHIFTERS — BENCH TEST

Wire 74HCT245 ×3 between ZIF socket and Teensy FlexIO pins. Load firmware that serves the original EPROM binary from flash. Use logic analyzer on address/data lines — verify Teensy responds to reads within 150ns timing window. Vary address lines with signal generator, confirm correct data returned. No ECU connected yet.

04

SENSOR INPUTS — BENCH TEST

Wire GM 3-bar MAP through 10kΩ/20kΩ divider to ADC1. Apply 5V to MAP Vcc. Vary vacuum port — confirm ADC readings change. Build knock bias circuit: 100kΩ/100kΩ voltage divider for 1.65V, 0.1µF coupling cap, 100pF RF filter. Confirm ADC2 reads ~1.65V at rest with no knock signal. Wire TPS and IAT tee-offs through dividers, confirm readings.

05

INJECTOR MOSFET STAGE — BENCH TEST

Wire IRLZ44N gates to Teensy D2–D5 via 100Ω gate resistors. Use a 12V LED as injector load substitute. Generate test pulse from Teensy — confirm gate switching with oscilloscope. Verify clean switching with no ringing. Confirm 1N4007 flyback correctly placed (cathode to 12V rail). Test pulse width trimming by varying gate delay in firmware — verify on scope.

06

FIRST ECU CONNECTION — ENGINE OFF

Install ZIF socket in ECU (original EPROM removed). Connect breadboard via ribbon cable through level shifters. Disconnect fuel pump relay first. Ignition on — logic analyzer should show ECU initiating address reads within milliseconds. Teensy responds with EPROM image data. Verify no address mismatches. No smoke = proceed.

07

FIRST START — EPROM EMULATION ONLY

Restore fuel pump relay. Start engine with Teensy serving stock EPROM image — zero corrections active. Engine should run identically to before. If it doesn't start or runs rough, EPROM emulation has a timing issue — return to bench. Once confirmed running normally, the foundation is solid. All correction layers added incrementally from here.

08

WIDEBAND + CLOSED LOOP — ACTIVATE

With engine running, enable Lambda reading from Spartan via UART. Datalog AFR vs RPM/MAP for a full drive cycle in open loop (no correction active) to understand where the stock tune is rich/lean on the 2.6L. Then enable injector trim with conservative ±10% limit. Expand limits once behavior is understood. This is where the 20× ADV resolution starts earning its keep.

09

Open KiCad (free). Transfer schematic from breadboard to KiCad schematic editor. Layout PCB with Teensy on removable pin headers, 74HCT245 in DIP sockets, LM2596 as plug-in sub-module, screw terminals for all external connections.
