Injector Signal Intercept
Cut ECU injector output wire. ECU side → IRLZ44N drain. Injector → IRLZ44N source. Teensy GPIO → gate via 100Ω resistor.

Teensy reads ECU's intended pulse timing on a digital input, then re-fires the MOSFET gate with corrected pulse width. Result: ±30% injector trim, wideband-referenced, RPM and MAP aware.

1N4007 flyback across each injector is mandatory — injector coil collapse will destroy the MOSFET without it.
MAF Signal Synthesis
Stock Hitachi MAF on the 7A/NF outputs a frequency signal (Hz proportional to airflow), not a DC voltage. Verify this with a scope before assuming.

Teensy reads original MAF freq on interrupt pin → scales it by a correction factor derived from MAP + displacement ratio (2.6L vs 2.3L stock) → outputs corrected frequency to ECU MAF input pin.

The ECU is "told" the engine is smaller than it is, causing it to request less fuel. The injector trim layer then adds the correct fuel back based on wideband feedback.
Spartan I2C Integration
The Spartan 2 OEM I2C puts Lambda data directly on the I2C bus. Teensy polls it at ~10ms intervals — faster than the sensor's 150ms response time, so no data is ever missed.

Wire.requestFrom(0x01, 2) — returns a 16-bit integer. Divide by 10000.0 to get Lambda float.

Compare to target Lambda table (indexed by RPM + MAP load), compute error, feed to injector trim PID loop. This is clean closed-loop fueling with zero analog noise.

ADC0 is now free — use for oil pressure or flex fuel sensor.
Knock Sensor Signal Conditioning
The Bosch knock sensor outputs a small AC voltage centered around 0V. Teensy ADC cannot handle negative voltages (0–3.3V only).

Required circuit:
• 0.1µF coupling cap in series (blocks DC)
• 100kΩ from signal to 1.65V bias voltage
• 1.65V from two 100kΩ resistors in series (3.3V divider)
• 100pF cap to GND for RF filtering
At 600MHz, Teensy can run a real-time FFT on the knock signal sampled at ~100kHz. Knock frequencies for the 2.3/2.6 5-cylinder are typically 6–9kHz.

When FFT energy at the knock frequency exceeds threshold, Teensy writes a modified ignition map entry via the EPROM emulation layer, pulling 1–3° of timing until knock subsides.

This is genuinely better than the stock ECU's crude knock threshold.

BUILD SEQUENCE
01

EPROM DUMP — DO THIS FIRST

Before touching anything, read the original EPROM out of the ECU with a TL866II+ or T48 programmer. This is your golden backup. If anything goes wrong, you reflash the original. Do not skip this step. Save the binary file somewhere safe — cloud backup, multiple USB drives.

02

POWER STAGE — BENCH TEST

Build the LM2596 module + AMS1117 circuit on a breadboard. Connect 12V bench supply. Measure output before connecting anything. You want exactly 5.00V ±0.05V and 3.30V ±0.05V. Add 0.1µF ceramics at Teensy VIN and GND. Power the Teensy. It should enumerate as USB device on your laptop.

03

EPROM EMULATOR — BENCH TEST

Wire the 74HCT245 level shifters between the ZIF socket and Teensy FlexIO pins. Load test firmware that simply serves the original EPROM binary from Teensy flash. Insert ZIF into a spare ECU socket (or use a test jig). With a logic analyzer, verify the Teensy is responding to address reads correctly at the right timing. Do NOT connect to the car ECU until this passes.

04

SPARTAN WIDEBAND — BENCH TEST

Wire Spartan 2 OEM I2C SDA/SCL to Teensy pins 18/19. Power Spartan from 12V (with its own inline fuse). Install the bung in the exhaust before this step — it needs to be in exhaust gas to read correctly. Run the I2C scanner sketch to confirm it appears at 0x01. Then test Lambda readback — with sensor in free air you should see ~1.0 Lambda or slightly lean.

05

SENSOR INPUTS — BENCH TEST

Wire GM 3-bar MAP sensor through the 10k/20k voltage divider to ADC1. Apply 5V to MAP sensor Vcc. Vary the vacuum/pressure port (use your mouth or a hand pump) and verify ADC readings change sensibly. Repeat for knock signal conditioning circuit — verify bias voltage is sitting at 1.65V with no signal applied.

06

INJECTOR MOSFET INTERCEPT — BENCH TEST

Wire IRLZ44N gates to Teensy D2–D5. Connect a 12V LED (or bench injector) as the load. Simulate ECU injector signals with a signal generator or Teensy itself. Verify Teensy can intercept and retime the pulse width accurately. Check 1N4007 flyback is correctly placed. Test with oscilloscope — you should see clean gate switching with no ringing.

07

FIRST ECU CONNECTION — ENGINE OFF

Remove original EPROM from ECU. Install ZIF socket. Connect breadboard via ribbon cable. Disconnect fuel pump relay first (so engine cannot start accidentally). Turn ignition on — ECU should power up and attempt to read the EPROM. Logic analyzer should show address reads happening. Verify Teensy is responding. No smoke = success.

08

FIRST START — BASIC EPROM EMULATION ONLY

Restore fuel pump relay. Start engine with Teensy serving the stock EPROM image only — no corrections active yet. Engine should run exactly as before. If it doesn't, the EPROM emulation has a timing issue. Once confirmed running, you can begin activating the sensor and correction layers one at a time.

09

PCB DESIGN

Once all breadboard stages work reliably, transfer to KiCad. Design a single compact PCB that mounts the Teensy 4.1 on pin headers (so it's removable), the 74HCT245 ICs in DIP sockets, the LM2596 module as a sub-board, and screw terminals for all external connections. 2-layer PCB from JLCPCB — 5 boards for ~$8. This becomes your prototype PCB kit.
